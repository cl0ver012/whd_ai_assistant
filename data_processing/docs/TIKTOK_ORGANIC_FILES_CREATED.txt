Quick Start
-----------
1. Run SQL setup:
   data_processing/sql/setup_tiktok_organic.sql

2. Test setup (optional):
   python data_processing/tests/test_setup_tiktok_organic.py

3. Process data:
   python data_processing/scripts/process_tiktok_organic.py

Data Location
-------------
Expected CSV files: NBX/TikTok Organic/
File pattern: TikTok {Month} {YY}.csv
Example: TikTok Apr 25.csv, TikTok Dec 24.csv

Features
--------
✅ All CSV columns preserved as database columns
✅ No embeddings (fast processing)
✅ No JSONB (direct column access)
✅ Batch processing (100 rows at a time)
✅ Duplicate detection (skips existing records)
✅ Full-text search support
✅ Optimized indexes for fast queries

Database Table: tiktok_organic
-------------------------------
Columns:
- id, created_at (system)
- file_name, period, year, month, month_name (metadata)
- date, day_of_month (date info)
- video_views, profile_views, likes, comments, shares (metrics)
- total_engagement, has_views, has_engagement (computed)
- content (search text)
